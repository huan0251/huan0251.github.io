Features of Boostrapt

Starter template

When starting an html, be sure to have your pages set up with the latest design and development standards. That means using an HTML5 doctype and including a viewport meta tag for proper responsive behaviors. You can go to Boostrap Website to find the latest start template then copy it and paste on your HTML file to start.

Mobile First

Bootstrap is developed mobile first, they optimize code for mobile devices first and then scale up components as necessary using CSS media queries. Be sure to add the responsive viewport meta tag to your <head> in HTML

Intuitive Components

Boostrap have many components, such as alents, buttons,cards, navbars, and popovers. These are simillar as CSS but the easier way with adding class into HTML.

JavaScript behavior navs and tabs

In the original,Tabs' navigation takes the basic nav and adds the .nav-tabs class to generate a tabbed interface. Here we use role="tablist", role="tab", role="tabpanel", and additional aria- attributes in order to convey their structure, functionality and current state to users of assistive technologies

As a best practice, The Boostrap website recommend using <button> elements for the tabs, as these are controls that trigger a dynamic change, rather than links that navigate to a new page or location.

See the example of this nav-tab, looks like a kind of cards.

Buttons

Bootstrap includes several predefined button styles, each serving its own semantic purpose, with a few extras thrown in for more control.

Button use .btn classes

When using button classes on <a>elements that are used to trigger in-page functionality (like collapsing content), rather than linking to new pages or sections within the current page, these links should be given a role="button" to appropriately convey their purpose to assistive technologies such as screen readers.

We use this concept on this navs and tabs example. If you want to see more usages, please visit the Boostrap website.

Carousel

The carousel is a slideshow for cycling through a series of content, built with CSS 3D transforms and a bit of JavaScript. It works with a series of images, text, or custom markup. It also includes support for previous/next controls and indicators.
